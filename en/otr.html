guitarProjection

developed for Toca (2024)

Code for the lights/projection performer of Toca.

It allows mapping standardized linear paths that the piece's spotlights will follow prior to the performance. Using a controller and the infrared camera image as feedback, the performer can choose the order of paths to use according to each moment of the piece, being able to modify in real time the horizontal position of each light relative to each path, a vertical offset, rotation, and transposition of the paths to adjust to the guitarist's performance. It is also possible to change the color of the lights, the size of their focus, their opacity, etc.

Developed in Python and Processing 4. Documentation for this code can be found here.

caminhooculto

developed for the SomaRumor residency (2021)

Interactive system based on the game Caminho Oculto. Instead of pieces of different shapes that must be moved to remove a specific piece, caminhooculto uses 8 LDRs and plastic pieces with different transparencies to identify randomized patterns on their surface. Each pattern controls distinct audio processing, also randomized from time to time.

Developed in Arduino and SuperCollider. Documentation can be found here.

joyOSCMapper

viewer and mapper of joycon inputs to OSC (2021)

Viewer and mapper of Nintendo Switch Joycon inputs to OSC, allowing integration with various interactive systems.

Developed in C++/openFrameworks for the piece digdigjoy, which hasn't (yet) left the drawing board. Here you can find the documentation.
